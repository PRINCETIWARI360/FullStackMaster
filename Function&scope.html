JavaScript Functions

Functions are reusable blocks of code that perform a specific task. They are fundamental building blocks in JavaScript, allowing you to encapsulate logic, avoid repetition, and create modular code.

Key Concepts

Functions can be declared in several ways: function declarations, function expressions, and arrow functions. Each has its own characteristics regarding hoisting, this binding, and syntax.

Function Types

Function Declaration

Declared using the function keyword. Hoisted to the top of their scope.

function calculateArea(width, height) {   return width * height; }
console.log(calculateArea(5, 10)); // 50

Function Expression

Assigned to a variable. Not hoisted like declarations.

const calculateArea = function(width, height) {   return width * height; };
console.log(calculateArea(5, 10)); // 50

Arrow Function

Shorter syntax, no this binding. Great for callbacks.

const calculateArea = (width, height) => width * height;
console.log(calculateArea(5, 10)); // 50

Parameters vs Arguments

Parameters are variables listed in the function definition. Arguments are the actual values passed to the function when it is called.

Scope in JavaScript

Scope determines the accessibility (visibility) of variables. JavaScript has three types of scope: global scope, function scope, and block scope (introduced with ES6).

Global Scope
globalVar

Accessible everywhere in your code

Function Scope
functionVar

Accessible only within the function

Block Scope
blockVar

Accessible only within the block (if, for, while, etc.)

const globalVar = 'Global'; // Global scope

function exampleFunction() {   const functionVar = 'Function'; // Function scope
  if (true) {     const blockVar = 'Block'; // Block scope     console.log(globalVar); // Accessible     console.log(functionVar); // Accessible   }
  // console.log(blockVar); // Error - not accessible }
exampleFunction(); console.log(globalVar); // Accessible // console.log(functionVar); // Error - not accessible

Variable Shadowing

When a variable in a local scope uses the same name as a variable in an outer scope, it "shadows" the outer variable. The inner variable takes precedence within its scope.

Closures

A closure is a function that remembers its outer variables and can access them. In JavaScript, all functions are naturally closures.

Global Scope
createCounter
Closure Scope
count
counter
function createCounter() {   let count = 0; // Local variable
  return function() {     count++; // Accesses count from outer function     return count;   }; }

const counter = createCounter(); console.log(counter()); // 1 console.log(counter()); // 2

Why Closures Matter

Closures are essential for:

  • Data encapsulation and privacy
  • Creating function factories
  • Implementing modules in JavaScript
  • Event handlers and callbacks
