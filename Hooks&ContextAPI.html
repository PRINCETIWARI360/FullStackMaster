React Hooks

Hooks are functions that let you "hook into" React state and lifecycle features from function components. They solve problems of reuse and organization in complex components.

useState Hook

// Import useState hook import { useState } from 'react'; function Counter() { // Declare state variable 'count' with initial value 0 const [count, setCount] = useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)}> Increment </button> </div> ); }

useEffect Hook

// Import useEffect hook import { useEffect } from 'react'; function UserProfile(props) { const [user, setUser] = useState(null); // Fetch user data when component mounts useEffect(() => { fetchUserData(props.userId) .then(data => setUser(data)); // Cleanup function return () => { // Cancel any ongoing requests }; }, [props.userId]); // Only re-run if userId changes return ( <div> {user ? <UserInfo user={user} /> : <p>Loading...</p>} </div> ); }

Common Hooks

  • useState: For managing component state
  • useEffect: For side effects (API calls, subscriptions)
  • useContext: For accessing context values
  • useReducer: For complex state logic
  • useCallback: For memoizing functions
  • useMemo: For memoizing values
  • useRef: For accessing DOM elements and mutable values

Context API

Context provides a way to pass data through the component tree without having to pass props down manually at every level. It's perfect for global data like themes, user authentication, or shopping carts.

Creating Context

// Create a Context import { createContext } from 'react'; // ThemeContext.js const ThemeContext = createContext({ theme: 'light', toggleTheme: () => {} }); export default ThemeContext;

Context Provider

// App.js - Providing context import { useState } from 'react'; import ThemeContext from './ThemeContext'; function App() { const [theme, setTheme] = useState('light'); const toggleTheme = () => { setTheme(prev => prev === 'light' ? 'dark' : 'light'); }; return ( <ThemeContext.Provider value={{ theme, toggleTheme }}> <Header /> <MainContent /> <Footer /> </ThemeContext.Provider> ); }

Consuming Context

// Header.js - Consuming context import { useContext } from 'react'; import ThemeContext from './ThemeContext'; function Header() { const { theme, toggleTheme } = useContext(ThemeContext); return ( <header className={theme}> <button onClick={toggleTheme}> Toggle Theme </button> </header> ); }

Best Practices

Hooks Rules

Only call hooks at the top level and from React functions.

Context Structure

Create separate contexts for different concerns (theme, user, cart).

Performance

Memoize context values and components with React.memo to prevent unnecessary re-renders.

Custom Hooks

Extract component logic into reusable custom hooks.

Combine with Reducers

Use useReducer with context for complex state management.

Dependency Arrays

Always specify correct dependencies in useEffect to avoid bugs.
