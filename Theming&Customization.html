Bootstrap Theming & Customization

Personalize Bootstrap to match your brand identity

CSS Variable Theming

Bootstrap 5 makes extensive use of CSS custom properties (variables) for its design system. You can override these variables to create your own theme.

Color Customization

Change the core theme colors by overriding the primary and secondary variables:

:root {
  --bs-primary: #6f42c1;
  --bs-secondary: #6610f2;
  --bs-success: #198754;
  --bs-info: #0dcaf0;
}

Component Customization

Customize specific components using CSS variables:

:root {
  --bs-border-radius: 0.5rem;
  --bs-box-shadow: 0 0.5rem 1rem rgba(0, 0, 0, 0.15);
  --bs-btn-border-radius: 0.5rem;
  --bs-card-border-radius: 0.5rem;
}

SASS Customization

For advanced customization, Bootstrap's SASS variables allow you to modify the source files before compilation.

Modifying SASS Variables

Create a custom SCSS file to override Bootstrap's defaults:

// Custom.scss
// Option A: Include all of Bootstrap

@import "../node_modules/bootstrap/scss/bootstrap";

// Then add overrides below
$primary: #6f42c1;
$secondary: #6610f2;
$border-radius: 0.5rem;
$box-shadow: 0 .5rem 1rem rgba($black, .15);

Creating Custom Utilities

Extend Bootstrap with your own utilities using SASS:

// Custom utilities
$custom-theme-colors: (
  "custom-primary": $primary,
  "custom-secondary": $secondary,
  "custom-light": $gray-100
);

// Merge with default theme colors
$theme-colors: map-merge($theme-colors, $custom-theme-colors);

// Generate utilities
@each $key, $val in $custom-theme-colors {
  .bg-#{$key} {
    background-color: #{$val} !important;
  }
}

Component Customization

Customize Bootstrap components by overriding their styles or creating new variations.

Custom Buttons

Create custom button styles with your theme colors:

.custom-btn {
  background-color: var(--custom-primary);
  border-radius: var(--custom-radius);
  box-shadow: var(--custom-shadow);
  transition: all 0.3s ease;
}

.btn-outline-custom {
  border-color: var(--custom-primary);
  color: var(--custom-primary);
}
