React Components & Props

Building Reusable UI with React

React Components

Components are the building blocks of any React application. They let you split the UI into independent, reusable pieces.

Functional Components

// Simple functional component function Welcome() { return <h1>Hello, React!</h1>; } // Using the component function App() { return ( <div> <Welcome /> </div> ); }

Class Components

// Class component class Welcome extends React.Component { render() { return <h1>Hello, React!</h1>; } } // Using the component class App extends React.Component { render() { return ( <div> <Welcome /> </div> ); } }

Component Types

  • Functional Components: Simple functions that return JSX
  • Class Components: ES6 classes that extend React.Component
  • Presentational Components: Focus on how things look
  • Container Components: Focus on how things work
  • Higher-Order Components: Functions that take a component and return a new component

React Props

Props (short for properties) are how components talk to each other. They are read-only and passed from parent to child components.

Passing Props

// Functional component with props function Welcome(props) { return <h1>Hello, {props.name}</h1>; } // Using the component with props function App() { return ( <div> <Welcome name="Sarah" /> <Welcome name="John" /> </div> ); }

Props with Class Components

// Class component with props class Welcome extends React.Component { render() { return <h1>Hello, {this.props.name}</h1>; } } // Using the component class App extends React.Component { render() { return ( <div> <Welcome name="Sarah" /> <Welcome name="John" /> </div> ); } }

Key Props Concepts

  • Props are read-only: Components should never modify their own props
  • Default props: Provide default values for props
  • PropTypes: Type checking for props
  • Children prop: Access content between component tags
  • Destructuring props: Cleaner way to access props

Best Practices

Component Design

Create small, focused components that do one thing well. Follow the Single Responsibility Principle.

Prop Naming

Use descriptive names for props. Prefix boolean props with "is" or "has" (e.g., isActive).

Immutable Props

Never mutate props directly. Use state for data that changes over time.

PropTypes

Use PropTypes to validate props and catch errors during development.

Destructuring

Destructure props at the top of your component for cleaner code.

Component Composition

Prefer composition over inheritance for code reuse and flexibility.

Advanced Concepts

Default Props

// Setting default props function Welcome(props) { return <h1>Hello, {props.name}</h1>; } Welcome.defaultProps = { name: 'Guest' }; // Using without name prop <Welcome /> // Renders "Hello, Guest"

PropTypes

// Import PropTypes import PropTypes from 'prop-types'; function UserCard(props) { // Component implementation } UserCard.propTypes = { name: PropTypes.string.isRequired, age: PropTypes.number, email: PropTypes.string.isRequired, isAdmin: PropTypes.bool };

Children Prop

// Component using children prop function Card(props) { return ( <div className="card"> {props.children} </div> ); } // Using the Card component <Card> <h2>Card Title</h2> <p>Card content goes here</p> </Card>
